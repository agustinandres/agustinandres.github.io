Financial Operations Management

This automation aims to optimize the monitoring and recording process of stock purchase and sale transactions by utilizing notifications sent by banks via their email service (Outlook). The automation workflow is designed to operate autonomously, perform financial performance calculations, and comply with the reporting regulations of the CNMV (National Securities Market Commission). Below is a detailed description of the process:

1. Daily Reading of the Outlook Mailbox

Every day, the program accesses the Outlook mailbox to search for emails from banks. These emails contain confirmations of stock purchase and sale transactions. The notifications can arrive in different formats:

  • PDF
  • Excel
  • Email body

2. Data Extraction from Notifications

When the program detects an email from a bank, it extracts the relevant transaction data, which may be in one of the three formats mentioned above (PDF, Excel, or email body). Depending on the format, the program implements different data extraction techniques:

  • For PDF files, the program uses a combination of OCR (Optical Character Recognition) and regular expressions for accurate data extraction.
  • For Excel files, the program does not rely on a fixed format. Instead, it applies a flexible logic to search for key cells anywhere in the file. This flexibility ensures that the automation continues to function even if banks modify their Excel templates.
  • For data in the email body, the program employs HTML analysis techniques to identify and extract data tables within the email. This is done through a parsing process that analyzes the hierarchical structure of HTML tags, locates the tables, and extracts the cell values accurately. Additionally, regular expressions are used to identify specific text patterns that may not be in tabular format, enabling more comprehensive data extraction even when the information is unstructured.

3. Data Entry into a Record Excel File

Once the program extracts the transaction data, it records them in an Excel file. This file serves as the central log of all completed transactions, organizing the data into relevant columns, such as:

  • Transaction date
  • Bank
  • Transaction type (buy/sell)
  • Number of shares
  • Share price
  • Total transaction value
  • Other relevant details

4. Automated Financial Calculations

The program performs a series of calculations on the extracted and recorded data:

  • Current Portfolio After Transactions: Calculates the portfolio value after each transaction, updating the number of shares and total value.
  • Yield Achieved: Determines the return generated by the transactions by comparing the purchase and sale prices of the shares.
  • % of Share Capital in Treasury Stock: Calculates the percentage of shares held as treasury stock in relation to the total outstanding shares.

5. XML Generation for CNMV Reporting

If the % of share capital in treasury stock exceeds a predefined threshold, the program automatically generates an XML file containing a history of all transactions performed since the last XML file was generated. This XML file follows a standardized format and includes the necessary information to comply with CNMV requirements.

6. XML Upload to the CNMV Website

Once the XML is generated, the automation accesses the CNMV website and logs into the appropriate section to upload the XML file. The process is carried out automatically without manual intervention, ensuring that the report is submitted within the required deadlines.

Benefits of Automation

  • Efficiency: The program executes the process automatically every day, eliminating the need for manual intervention.
  • Accuracy: Data extraction and financial calculations are performed precisely, minimizing human error.
  • Regulatory Compliance: The automation ensures that transactions are correctly reported to the CNMV within the established deadlines, meeting regulatory requirements.
  • Time Savings: Automating this process saves hundreds of hours of manual work, allowing financial analysts to focus on more strategic tasks.

Calypso-Refinitiv Rate Control

This automation aims to ensure that the database in the Calypso system always contains up-to-date market information, particularly regarding exchange and interest rates. By automating this process, it guarantees that crucial data for financial decision-making is always current, reducing the risk of using outdated information and improving operational efficiency within the company. Below is a detailed workflow of the automation:

1. Accessing Calypso and Reviewing Exchange and Interest Rates

The process begins with accessing the Calypso system to review the registered exchange and interest rates. If any data is found with a date prior to the last business day, it is automatically added to an update list, ensuring that values remain up-to-date for operations.

2. Logging into Refinitiv

The automation then accesses Refinitiv’s Datascope platform using the corresponding credentials. To ensure data security, access credentials are not stored in the program’s source code but are encrypted within a secure environment.

3. Retrieving Exchange Rates from Refinitiv

With the missing data identified in Calypso, the automation performs an efficient search in Refinitiv to obtain the most up-to-date values. This process is carried out swiftly and autonomously, ensuring no valuable time is wasted on data collection.

4. Generating XML Files

Once Refinitiv provides the necessary data, the program automatically generates XML files containing the exchange and interest rate information. These files are ready for upload without requiring manual intervention.

5. Automatic Upload of XML Files to MFT

The generated XML files are automatically uploaded using the Managed File Transfer (MFT) platform, employing robust web scraping techniques to interact with the MFT web interfaces. This ensures reliable and accurate file uploads without manual intervention.

6. Retrieving XML Files from Alveo

After the XML files are sent via MFT, the system accesses Alveo to locate them in the designated path where MFT stored them. It then loads the data from these XML files, ensuring all necessary information is available for the next phase of the process.

7. Transferring Data to Calypso

With the data successfully loaded into Alveo, the automation transfers the information from Alveo to Calypso, updating exchange and interest rates in a fully automated manner.

8. Final Review in Calypso

Finally, the process verifies in Calypso that the data has been correctly updated. This ensures there are no discrepancies and that systems remain ready for operations with accurate and up-to-date information.

Benefits of Automation

This process not only ensures data accuracy but also provides several additional benefits for the company:

  • Operational Efficiency: Automating this process eliminates manual intervention and associated errors, ensuring updates are performed accurately and in real time.
  • Cost Reduction: By removing manual tasks, the automation optimizes resource utilization, leading to significant savings in time and effort. Hours spent on repetitive, error-prone tasks can now be allocated to higher-value activities.
  • Regulatory Compliance: The process ensures that all exchange and interest rates are always up-to-date and aligned with internal and regulatory requirements, reducing the risk of penalties or reporting errors.
  • Time Savings: By automating tasks that previously required manual intervention, the process becomes much more efficient, allowing financial teams to focus on strategic tasks and value-added analysis.
  • Scalability: The automation is designed to adapt to future data update needs, allowing seamless scaling according to business requirements.

With this automated system, the updating of exchange and interest rates is fast, accurate, and requires no manual intervention, enabling the company to remain competitive and aligned with the highest industry standards.
